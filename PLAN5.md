# 三端全流程视觉验收微调计划

## Summary
- 下一步优先做微信开发者工具里的视觉验收微调，覆盖老人端、子女端、管理员端。
- 验收设备按 `iPhone 12/14 + 一个常见安卓尺寸`。
- 旧模板、旧 README、quickstart 云函数残留本轮先不删除，只做标注说明。

## 你需要先提供
- 微信开发者工具截图，建议每个设备尺寸各一套。
- 至少包含：身份选择、老人注册、首页、测评、每日练地图/学习/练习、我的、训练记录、健康档案、子女端首页、管理员端首页。
- 如果某些页面已经明显满意，可以只提供问题页面，并说明“其他页面暂不动”。

## Key Changes
- 视觉微调：根据截图修正首页视频号卡片、注册页头像比例、身份选择页角色卡片大小，以及三端页面中出现的文字溢出、按钮遮挡、间距不均、底部区域拥挤等问题。
- 三端一致性：老人端保持适老大字和低步骤；子女端/管理员端保持 mock 工作台风格，不引入真实登录、云端或编辑保存能力。
- 工程标注：更新项目状态或 README，说明 `pages/index`、`pages/example` 空目录、云开发 quickstart 残留和当前主工程边界；不删除这些残留。

## Interfaces
- 不新增路由，不改变 `app.json` 页面结构。
- 不改变 `storage.js` 的本地存储 key。
- 不改变 mock 数据结构，除非截图暴露必须补充的展示文案或占位数据。

## Test Plan
- 在微信开发者工具中按 iPhone 12/14 和常见安卓尺寸跑通三端入口。
- 老人端完整检查：身份选择 -> 注册 -> 首页 -> 测评 -> 每日练 -> 我的 -> 训练记录/健康档案 -> 回身份选择。
- 子女端和管理员端检查页面首屏、主要卡片、按钮反馈和返回路径。
- 静态检查：JS 语法、JSON 解析、`app.json` 路由对应页面文件完整。

## Assumptions
- 当前版本仍是本地 mock 演示版。
- 真实语音识别、真实登录、云数据库、真实姿态识别不进入本轮。
- 截图是视觉修改的最高优先依据；没有截图的页面只做低风险一致性微调。
